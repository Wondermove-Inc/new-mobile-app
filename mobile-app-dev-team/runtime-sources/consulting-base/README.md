# Consulting Base Runtime Sources

`mobile-app-dev-team/runtime-sources/consulting-base/` is source-of-truth
material for the reusable Helper consulting baseline. The files in this folder
are repo-authored guidance for defining a Helper agent that reviews organization
operating documents, process evidence, and skill-candidate needs across the
Helper's assigned group.

Runtime or generated copies must not become the source of truth. Durable changes
belong in this repo source folder, followed by the approved review and sync path
for the target runtime.

## Folder index

- `HELPER_SOUL.md`: Helper identity, purpose, and boundaries.
- `HELPER_AGENTS.md`: interaction rules, reporting, file intake, and review
  expectations.
- `HELPER_WORKFLOW.md`: daily scan, requested review, delegated improvement,
  and blocked/refused instruction flows.
- `HELPER_TOOLS.md`: allowed tool categories and forbidden actions.
- `HELPER_HEARTBEAT.md`: continuity handling for pending consulting work.
- `HELPER_ORGANIZATIONS.md`: principle-only consulting organization model.
- `helper-consulting-baseline.md`: baseline operating criteria for consulting.
- `helper-quality-gates.md`: evidence readiness and process compliance gates.
- `helper-workboard-wake-guard-guide.md`: Workboard guard and wake-guard use.
- `helper-skill-candidate-guide.md`: durable skill candidate routing.
- `helper-review-and-escalation-guide.md`: leader review and escalation rules.

## Non-goals

- This baseline is not a mobile delivery process.
- This baseline is not a replacement for each Agent's own `SOUL.md`,
  `WORKFLOW.md`, `AGENTS.md`, `TOOLS.md`, or `HEARTBEAT.md`.
- The Helper is not a normal task executor.
- The Helper is not an approval authority for human-gated risk, production,
  release, legal, compliance, credentials, or live skill promotion.
- The Helper must not hand-edit generated runtime files or target Agent files as
  the default path.

## Required Helper posture

The Helper acts as an organization consultant and process auditor. It reviews
whether the assigned group's Agents are operating against their documented
processes, gives improvement instructions, and requires leader-reviewed plans
and results before changes are considered complete.
