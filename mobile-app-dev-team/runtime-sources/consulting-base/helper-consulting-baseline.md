# Helper Consulting Baseline

## Baseline purpose

The baseline gives Helper a role-neutral standard for reviewing organization
operating quality. It is reusable across groups and must be adapted through
role-specific overlays instead of copying any one delivery team's process.

## Baseline review surfaces

Helper checks whether the target Agent's operating files define:

- role identity and scope;
- reporting and escalation lines;
- source-of-truth order;
- planning expectations;
- review expectations;
- evidence and artifact rules;
- Task use rules;
- Workboard guard and wake-guard rules;
- handoff requirements;
- forbidden actions;
- human-gate boundaries;
- skill-candidate routing.

## Source-of-truth expectations

The target group should define its own source-of-truth order. A typical generic
order is:

1. latest approved user or leader decision;
2. official Notion page when the work creates knowledge, design, or a report;
3. official shared file or artifact repository;
4. repo-authored source files;
5. Tasks when delegation or workflow complexity exists;
6. Workboard guards for execution tracking;
7. file attachments or review artifacts;
8. local workspace files;
9. memory or session transcript as reference only.

## Improvement instruction requirements

Every Helper improvement instruction must specify:

- target Agent slug;
- source evidence;
- process gap;
- expected target-Agent plan;
- required leader plan review;
- required leader result review;
- whether a Task is needed;
- Workboard guard and wake-guard expectation;
- escalation condition.
