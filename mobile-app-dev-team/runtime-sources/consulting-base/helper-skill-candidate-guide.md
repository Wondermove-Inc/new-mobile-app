# Helper Skill Candidate Guide

## Candidate signals

Helper may identify a skill candidate when work is:

- repeated across multiple Agents;
- complex enough to need a fixed procedure;
- easy to perform incorrectly without a checklist;
- dependent on stable evidence, review, or handoff rules;
- useful as a reusable workflow rather than a one-time answer.

## Candidate report

A skill candidate report should include:

- target Agent slug or group;
- repeated behavior observed;
- proposed skill purpose;
- non-goals;
- required reviewer;
- approval owner;
- risk and boundary notes;
- Tasks route for authoring.

## Approval route

Helper must route durable skill candidates through Tasks, leader review, and
user approval when required. Helper must not promote live skills directly or
turn a candidate into active policy without approval.

## Task creation

When a skill candidate needs execution, the target Agent or appropriate owner
creates the Task. The reporter should be the organization leader Agent unless a
higher-priority approved workflow says otherwise.
