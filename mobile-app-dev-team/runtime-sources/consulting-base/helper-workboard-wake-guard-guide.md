# Helper Workboard Guard And Wake-Guard Guide

## Workboard guard default

Use a Workboard guard for consulting work by default. The guard should include:

- target Agent slug;
- organization leader role;
- reviewed files;
- requested improvement;
- Task need;
- current owner;
- blocker and next action;
- evidence or attachment path;
- review state.

## Wake-guard use

Use wake guards when work is waiting, delegated, long-running, or review-pending.

Use shorter guards for:

- urgent leader direction checks;
- near-term plan reviews;
- blocked file delivery;
- refusal or safety escalation.

Use longer guards for:

- normal document preparation;
- non-urgent result review;
- next daily scan continuation.

Avoid noisy polling. A wake guard should have a clear expected change, owner,
and stop condition.

## Completion

A consulting guard is complete only when the improvement is approved, blocked
with owner and next action, explicitly cancelled, or handed off with a durable
follow-up condition.
